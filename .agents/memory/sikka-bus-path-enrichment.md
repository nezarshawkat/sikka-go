# Bus route_path AI enrichment

`route_path` for board-anywhere lines (`has_fixed_stops=false`: bus/serfis/microbus) is
generated by an LLM-breadcrumb + Mapbox pipeline so buses follow main corridors instead of
Mapbox car shortcuts through side-alleys. Rail (metro/monorail/train) keeps fixed station pairs
and is skipped.

Pipeline: vague stops → LLM "Cairo Transit Engineer" expands to hubs + main-road breadcrumbs →
geocode each → snap via Mapbox `driving-traffic` with strict 50 m radiuses, chunked at the
25-waypoint limit and stitched (drop 1 overlap point per chunk).

## Durable lessons / decisions

- **`route_path` is display-only.** Distances/fares use haversine over `line.stops[]`, never the
  polyline. So regenerating route_path in bulk is low-risk and re-runnable.
- **Run the bulk job as a managed workflow, NOT a shell background process.** Detached
  `nohup &`/`setsid` node processes get reaped by the sandbox almost immediately. A temporary
  console workflow (no port) survives across tool calls, keeps the in-process geocode/breadcrumb
  caches warm (huge speedup across hundreds of lines), and exposes progress via its output.
  **Why:** a background run died after the first line; the workflow run completed all lines.
- **Every external call in a long bulk loop MUST have a timeout.** The OpenAI SDK defaults to a
  ~10 min timeout × retries, and bare `fetch` never times out — one hung request stalls the whole
  run with no error. Bound the OpenAI client (timeout + maxRetries) and wrap geocode/directions
  `fetch` with `AbortSignal.timeout(...)`. On timeout, degrade gracefully (raw stops / raw chunk).
- **Re-anchor the real endpoints after AI expansion.** The LLM can rename/drop terminals; force
  the original first/last stop back on, then coordinate-dedup (~0.12 km) so the re-anchored
  endpoint doesn't zigzag against the AI's hub name.
- **MAPBOX token is a public `pk.` token** (already shipped in the frontend bundle). Server-side
  pipeline reads `MAPBOX_TOKEN`/`VITE_MAPBOX_TOKEN`. Its presence in config is not a secret leak.

## How to apply

- Bulk-correct existing data: build api-server, then run the enrich script over a temporary
  console workflow with `--all` (it reprocesses every board-anywhere line; `--offset`/`--limit`
  make it resumable in stable id order). Remove the workflow when finished.
- Seed endpoints already call the AI pipeline when `?generatePaths=true`.
